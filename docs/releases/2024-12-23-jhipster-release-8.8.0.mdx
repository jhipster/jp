# JHipster リリース v8.8.0

これはJHipster v8のマイナーリリースです。

これは[8.7.3リリース](/2024/11/01/jhipster-release-8.7.3.html)以降、メインブランチで[535件のクローズ済み課題とプルリクエスト](https://github.com/jhipster/generator-jhipster/issues?q=is:closed+milestone:8.8.0)が含まれています。

## 新機能

* Spring Boot 3.4へのアップグレード ([#27963](https://github.com/jhipster/generator-jhipster/pull/27963) および [#28249](https://github.com/jhipster/generator-jhipster/pull/28249))
* Spring Boot 3に対応したCouchbaseの修正 ([#27972](https://github.com/jhipster/generator-jhipster/pull/27972))
* Gradle 8.12へのアップグレード ([#28259](https://github.com/jhipster/generator-jhipster/pull/28259))
* Angular 19へのアップグレード ([#28114](https://github.com/jhipster/generator-jhipster/pull/28114))
* Angular向けの実験的esbuildサポートの追加 ([#28003](https://github.com/jhipster/generator-jhipster/pull/28003))
* React Router 7へのアップグレード ([#27985](https://github.com/jhipster/generator-jhipster/pull/27985))
* Node 22.12.0へのアップグレード ([#28080](https://github.com/jhipster/generator-jhipster/pull/28080))
* 言語プロンプトの修正 ([#28035](https://github.com/jhipster/generator-jhipster/pull/28035))
* SPAにおけるCSRFトークンハンドリングの改善 ([#27908](https://github.com/jhipster/generator-jhipster/pull/27908))
* テストヘルパーへのブループリントサポートの追加 ([#27768](https://github.com/jhipster/generator-jhipster/pull/27768))
* Angularジェネレータでのジェネリックスの使用 ([#27550](https://github.com/jhipster/generator-jhipster/pull/27550))
* TypeScript 5.7.2へのアップグレード ([#28213](https://github.com/jhipster/generator-jhipster/pull/28213))

## インストール方法

JHipster v8.8.0をインストールするには：

```
npm install -g generator-jhipster@8.8.0
```

ソースコードから自動的にビルドされるため、JHipster Dockerイメージでも利用できます。

- [JHipster Online](https://start.jhipster.tech)
- [JHipster Devbox](https://github.com/jhipster/jhipster-devbox)

## アップグレード方法

**自動アップグレード**

自動アップグレードには、既存のアプリケーションで[JHipsterアップグレードサブジェネレータ](/upgrading-an-application/)を使用します。

JHipsterのバージョンをアップグレードします：

```
npm update -g generator-jhipster
```

そして、アップグレードサブジェネレータを実行します：

```
jhipster upgrade
```

より高度なアップグレード機能には、[migrateブループリント](https://github.com/jhipster/generator-jhipster-migrate)も使用できます。

```
npm i -g generator-jhipster-migrate
jhipster-migrate
```

**手動アップグレード**

手動アップグレードには、まず以下のコマンドでJHipsterのバージョンをアップグレードします。

```
npm update -g generator-jhipster
```

既存のプロジェクトがある場合、生成されたJHipsterバージョンを引き続き使用します。
プロジェクトをアップグレードするには、まず`node_modules`フォルダを削除してから、以下を実行する必要があります。

```
jhipster
```

JHipster 8.0以降、このコマンドはプロジェクトとそのすべてのエンティティを更新します。

エンティティサブジェネレータを再度実行して、エンティティを1つずつ更新することもできます。たとえば、エンティティの名前が_Foo_の場合は、以下を使用します。

```
jhipster entity Foo --single-entity
```
